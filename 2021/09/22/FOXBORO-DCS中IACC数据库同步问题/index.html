# CP 内容 当前控制器中的运行内容

Upload 在 ICC 中使用的命令，从 CP 中同步实时数据并更新 ICC 数据库中数据。
将 ICC 数据库中包含的 blocks 与 CP 中相同的 blocks 进行参数更新，ICC 中不包含的且 CP 中包含的 BLOCKS 不会被同步。

# Upload and Compare To CP

先进行Upload更新ICC数据库，再进行IACC数据库内容与ICC数据库内容进行对比。

# Compare To CP

进行IACC数据库内容与ICC数据库内容进行对比。

正常情况下ICC数据库与CP中的BLOCKS数量是一致的，有特殊情况CP中的内容会比ICC数据库中的BLOCK多，导致控制器初始化下种后会丢失CP中多出的block块。

导致CP中block块多出的原因是当ICC处于编辑状态时异常崩溃就会出现ICC数据库中的BLOCKS丢失，CP中的并不会收到影响。

保存CP中完整内容的方法只有做checkpoint，对CP进行快照备份。

# 如何抢救 icc 数据库中丢失的内容？

1、 在 system management 中对控制器做 checkpoint。
2、 保存好 DBCPNAME.UC 文件，此文件为 CP 快照文件。
3、 在 icc 中添加缺少的 block 块，第一次提示错误确定后 CP 中的同名 block 块被冲掉，第二次继续新建缺失 BLOCK 块，建立成功。此时不用管 BLOCK 块的内容，只要名称和类型一致就可以。
4、 全部缺失的 block 块新建完毕后，恢复备份的 DBCPNAME.UC 文件后重启控制器。
5、 控制器启动完毕后，做 UPLOAD 操作，新建 block 块与 CP 中的 block 块同步完毕。
6、 恢复结束。

注：经过测试此方法只能恢复部分参数，CALCA 块的程序步骤和连接并不能恢复。
